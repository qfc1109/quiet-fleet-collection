# 轻帆集 / Quiet Fleet Collection 需求分析

## 项目定位

轻帆集是一个个人文档与项目笔记展示网站。第一版面向个人使用：公开访客无需登录即可浏览公开页面、项目、文档和文件；网站用户从主站登录/注册入口进入；后台管理员进入独立后台网站维护用户和权限。

项目后续会作为个人开源项目提交到 GitHub，并逐步扩展审核、操作日志、搜索、收藏、评论等能力。

## 第一版目标

- 提供公开首页，作为同事访问入口；第一版首页只保留站点介绍和进入项目广场的入口，不直接展示项目列表。
- 提供公开项目广场，展示别人公开的项目列表。
- 提供公开项目展示页，展示某个项目下的文件列表和文件内容。
- 提供主站登录页，支持网站用户登录和注册。
- 主站登录页只面向网站用户登录和注册，不提供后台管理系统跳转入口。
- 提供个人空间，网站用户登录后可以进入账号中心、创建项目、编辑项目、上传文件、管理文件。
- 提供独立后台管理最小版，管理网站用户、后台管理员账号和后台角色权限。
- 后台管理前端使用 Vue 3 + Element Plus 组件库实现，用户管理默认展示列表，新增和编辑通过按钮入口打开弹窗表单。
- 网站业务数据、网站日志、后台管理数据、后台日志分别保存到独立数据库。
- 文件归属到项目，组织方式类似 GitHub 仓库。
- 支持上传整个文件夹，保留项目内相对路径。
- 支持在项目管理中将文件移动到已有目录或项目根目录，便于重新上传文档后整理位置。
- Markdown 中引用的相对图片随同一文件夹上传后，应能在预览页正常显示。
- 支持 Markdown、PDF、图片、Excel 的基础在线预览，以及 Word 基础文本预览。
- 其他文件格式展示基础信息并提供下载。
- 登录态基于 Cookie Session 时，所有写请求必须带 CSRF Token；登录成功后必须更换 Session ID，避免会话固定。
- 上传头像和文件不能只信任客户端 `Content-Type`；头像按服务端识别的图片类型保存，内联文件响应禁止浏览器 MIME 嗅探。
- 项目公开路径 `slug` 和可见性 `visibility` 需要服务端校验，重复路径或非法可见性返回明确业务错误。
- 使用 Spring Boot + MySQL + Vue 3 技术路线。

## 安全与输入校验

- 前端通过 `/api/auth/csrf` 获取 CSRF Token，并在 `POST`、`PUT`、`DELETE` 等写请求中携带 `X-CSRF-Token`。
- 主站网站用户 session 和后台管理员 session 独立保存；后台管理员不能凭后台登录态进入主站个人空间创建或编辑个人项目。
- 网站用户和后台管理员登录成功后都需要刷新 Session ID。
- 用户输入默认使用 Vue 文本插值渲染；确实需要渲染 HTML 的地方必须经过 DOMPurify 等 HTML 清洗链路。
- 后端统一处理未知运行时异常，生产响应不暴露堆栈细节。

## 第一版角色

### 访客

访客不需要登录，可以访问：

- 首页
- 项目广场
- 项目展示页
- 已公开文件的预览和下载

### 网站用户

网站用户是使用主站的普通账号。第一版提供公开注册入口，登录后可以访问：

- 个人空间
- 账号资料和头像管理
- 项目创建和编辑
- 文件上传和管理

后台管理系统可以维护网站用户账号：

- 查看网站用户列表
- 创建网站用户
- 编辑显示名、简介和状态
- 重置密码
- 停用账号

第一版不实现网站用户的社交、评论、收藏等能力。

### 后台管理员

后台管理员可以进入后台管理系统。第一版通过角色权限控制后台能力：

- `SUPER_ADMIN`：拥有后台全部权限，默认绑定 `admin`
- `NORMAL_ADMIN`：普通管理员，维护网站用户、项目、文件和反馈，不管理后台账号权限

网站用户和后台管理员保存在不同数据库、不同表、不同登录态中，允许存在相同账号名。第一版默认在网站用户表和后台管理员表中各初始化一个 `admin / 123456`，两者互不代表对方。

## 页面范围

### 首页

首页公开访问，承担站点门面和入口职责。第一版可以简洁，但需要预留后续扩展空间。

主要内容：

- 网站名称：轻帆集 / Quiet Fleet Collection
- 简短介绍
- 进入项目广场的入口
- 后续站点介绍或视觉样式扩展空间

### 项目广场

项目广场公开访问，类似 GitHub 的公开项目入口。用户可以在这里看到别人公开出来的项目资料。

主要内容：

- 项目列表
- 项目公开入口
- 不额外展示公开项目数量或二级“公开项目”标题，直接呈现所有公开项目

### 项目展示页

项目展示页公开访问，类似 GitHub 仓库展示页。

主要内容：

- 项目名称
- 项目简介
- 项目文件列表
- 文件类型、大小、上传时间
- 文件预览区域
- 文件下载入口

后续功能记录：每个项目可以手动指定一个 Markdown 文件作为 README/首页文档。第一版不实现该能力。

### 登录页

登录页提供网站用户登录和注册。

主站登录页只处理网站用户登录和注册。后台管理系统通过独立地址访问，不在主站首页、主站导航或登录页提供跳转入口。

### 个人空间

个人空间需要网站用户登录后访问，不依赖后台 `PROJECT_MANAGE` 权限。

主要内容：

- 项目列表管理
- 创建项目
- 编辑项目基础信息
- 软删除/隐藏项目，并记录主站操作日志
- 上传单个文件、多个文件或整个文件夹到指定项目
- 将文件移动到已有目录或项目根目录
- 删除文件
- 查看公开展示页入口

### 后台管理

后台管理是独立后台网站，需要登录并拥有后台权限后访问，不作为主站路由展示。

第一版包含：

- 网站用户管理
- 后台管理员账号管理
- 角色权限管理

界面形态采用标准管理后台布局：左侧为模块路由栏，右侧为当前模块展示区域，顶部保留当前账号和退出操作，不提供跳转主站入口。

后台页面使用 Element Plus 组件库：

- 用户管理和管理员管理默认是列表页，主区域展示表格数据。
- 新增用户、新增管理员只作为按钮入口，不在列表页常驻大表单。
- 编辑账号、重置密码和保存结果通过弹窗、消息提示等管理后台常见控件完成。
- 角色权限模块使用表格、分区表单和复选框组管理固定角色权限。

后台管理页面必须优先符合真实后台系统的使用习惯，而不是展示型页面：

- 列表页只负责查询、筛选、浏览、分页、批量操作和进入后续操作。
- 新增、编辑、详情、日志、关联数据等能力通过明确按钮或操作列进入。
- 简单表单使用弹窗；字段多、步骤多或需要长期停留编辑的配置使用独立路由页面。
- 不把新增表单、编辑表单、详情面板和列表数据全部平铺在同一个页面中。
- 页面视觉保持克制、清晰、信息密度适中，避免大面积宣传页式卡片、大留白和装饰性布局。

角色权限是后台管理中的一个小模块，不作为公开网站功能展示。

## 文件预览范围

| 文件类型 | 第一版处理方式 |
| --- | --- |
| Markdown | 在线渲染，并解析同项目文件夹内的相对图片 |
| PDF | 浏览器内嵌预览 |
| 图片 | 在线预览 |
| Excel | 基础表格预览 |
| Word | 基础文本预览，明确提示图片和复杂版式需下载原文件查看，支持下载 |
| PPT | 展示文件信息，支持下载 |
| 压缩包 | 展示文件信息，支持下载 |
| 视频 | 展示文件信息，支持下载 |
| 其他格式 | 展示文件信息，支持下载 |

## 第一版不做

- 复杂权限菜单和细粒度数据权限
- 审核流程
- 操作日志
- 评论
- 收藏
- 全文搜索
- GitHub 版本管理
- 项目 README 首页文档绑定
- 视频在线播放
- Word 图片和复杂版式在线预览 / Word 高保真在线预览 / PPT 在线预览

## 后续规划

- 完善登录权限校验体系。
- 扩展后台管理系统，增加项目、文件、审核、业务配置和菜单权限。
- 增加操作日志，记录上传、删除、编辑、登录等行为。
- 增加项目 README/首页文档能力。
- 增加全文搜索、收藏、评论。
- 增加视频在线播放和更多办公文档预览能力。
- 增加 Docker 部署和云服务器部署文档。
- 初始化 GitHub 开源工程规范。
